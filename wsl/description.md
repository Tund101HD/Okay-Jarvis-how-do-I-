> *"Sir, you are running Linux on Windows. I have logged the paradox for posterity."* — J.A.R.V.I.S. (probably)

This directory collects guides centered around **WSL2** — setting it up, plugging it into IDEs, sharing toolchains between Windows and Linux without one of them eating the other's file permissions, and the small quality-of-life fixes that turn "technically working" into "actually pleasant."

If you're here, you're likely trying to:

- Develop on Linux without leaving your Windows machine.
- Hook a Windows IDE (CLion, WebStorm, VS Code, etc.) into a WSL toolchain.
- Figure out why something that works in PowerShell silently breaks the moment you cross the `\\wsl$` boundary.
